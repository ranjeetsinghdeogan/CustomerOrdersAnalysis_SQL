# Customer Orders Analysis using Microsoft SQL Server

## Project Overview

This project demonstrates the use of SQL joins and subqueries to analyze customer order data from the Northwind database. The goal was to practice combining multiple tables, performing aggregations, and extracting meaningful business insights using Microsoft SQL Server.

## What I Did

* Joined the **Customers** and **Orders** tables to analyze customer activity
* Calculated the total number of orders for each customer
* Identified customers who have not placed any orders
* Used a subquery to find orders with amounts greater than the average order amount
* Calculated total order amounts for each customer
* Retrieved the top 3 customers who spent the most

## How I Did It

* Used **Microsoft SQL Server Management Studio (SSMS)**
* Applied SQL concepts such as:

  * `JOIN` (INNER JOIN, LEFT JOIN)
  * `GROUP BY`
  * `COUNT()` and `SUM()` functions
  * `ORDER BY`
  * `TOP` keyword
  * Subqueries for comparison with average values

## Result

* Successfully combined data from multiple tables using joins
* Identified customers with and without orders
* Determined high-value orders compared to the average
* Ranked customers based on total spending
* Demonstrated practical understanding of SQL joins, aggregation, and subqueries

## Tools Used

* Microsoft SQL Server
* SQL Server Management Studio (SSMS)
* Northwind Database
